# ALMS 0.1.0

Release date: 2026-06-09

## Release Summary

`0.1.0` is the initial public release of ALMS as a small, self-hosted MCP server for shared agent memory.

This release establishes:

- the public project identity
- the initial MCP tool and resource surface
- the baseline deployment model
- the prompt and skill assets for agent integration
- the governance files required for open-source collaboration

## Highlights

- single-binary Go server
- PostgreSQL-backed learning and registry storage
- MCP-native transport for agent tooling
- gap-safe sync acknowledgement workflow
- protocol distribution by agent tag
- public documentation set under `documentation/`

## Known Boundaries

- shared-token auth is intentionally minimal
- sync semantics are usable but still need contract hardening
- enrichment/scoring workflow naming is not fully aligned across docs and server APIs

## Open Source Terms

ALMS `0.1.0` is released under the MIT license.

Why MIT:

- simple for outside adopters to understand
- permissive for research and commercial use
- low-friction for integration into broader agent platforms

## Upgrade Notes

This is the first public release, so there is no upgrade path from a prior tagged version.
