## When To Open An Issue

Open an issue when you have:

- a reproducible bug
- a documentation problem
- a clearly scoped feature request
- a compatibility question that is not answered by the docs

## What To Include

- ALMS version or commit
- environment details
- request or command used
- actual behavior
- expected behavior
- logs or error output when relevant
